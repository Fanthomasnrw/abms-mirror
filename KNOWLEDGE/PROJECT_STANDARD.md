# PROJECT STANDARD
## Asphalt Burn™ Project Governance Standard

**Version:** 1.0.0  
**Status:** RELEASED  
**Classification:** GOVERNANCE  
**Owner:** Thomas Ridderskamp

---

# Purpose

Dieser Standard definiert die Planung, Durchführung, Überwachung und den Abschluss aller Projekte innerhalb des Asphalt Burn™ Management Systems.

Jedes Projekt besitzt einen klar definierten Zweck, dokumentierte Ziele und nachvollziehbare Ergebnisse.

---

# Scope

Dieser Standard gilt für:

- Produktentwicklungen
- Marketingkampagnen
- Shop-Projekte
- IT-Projekte
- Automatisierungen
- Lieferantenprojekte
- Infrastrukturprojekte
- interne Organisationsprojekte

---

# Project Principles

Alle Projekte folgen den Grundsätzen:

- klare Ziele
- dokumentierter Umfang
- definierte Verantwortlichkeiten
- messbare Ergebnisse
- vollständige Nachvollziehbarkeit

---

# Project Identification

Jedes Projekt erhält eine eindeutige Projekt-ID.

Beispiele:

- ABMS-2026-BRAND-001
- ABMS-2026-SHOP-001
- ABMS-2026-MARKETING-001

Projekt-IDs werden niemals erneut vergeben.

---

# Mandatory Project Documentation

Jedes Projekt besitzt mindestens:

- Projekt-ID
- Projekttitel
- Ziel
- Beschreibung
- Owner
- Startdatum
- Status
- Meilensteine
- Risiken
- Ergebnisse
- Abschlussbewertung

---

# Project Lifecycle

Jedes Projekt durchläuft folgende Phasen:

1. Idea
2. Planning
3. Approval
4. Execution
5. Monitoring
6. Review
7. Closure
8. Archive

Keine Phase darf übersprungen werden.

---

# Milestones

Meilensteine definieren überprüfbare Zwischenergebnisse.

Jeder Meilenstein besitzt:

- Beschreibung
- Termin
- Status
- Verantwortlichen

---

# Risk Management

Für jedes Projekt werden Risiken dokumentiert.

Mindestens:

- Risiko
- Eintrittswahrscheinlichkeit
- Auswirkung
- Gegenmaßnahmen
- Verantwortlicher

---

# Project Review

Nach Abschluss erfolgt eine Bewertung.

Dokumentiert werden:

- Zielerreichung
- Abweichungen
- Lessons Learned
- Verbesserungspotenziale

---

# Runtime Reference

Operative Projektarbeit erfolgt ausschließlich innerhalb:

RUNTIME/

- PROJECTS/

---

# Governance Rules

1. Kein Projekt ohne Projekt-ID.
2. Jedes Projekt besitzt dokumentierte Ziele.
3. Risiken werden dokumentiert.
4. Änderungen werden nachvollziehbar festgehalten.
5. Projekte werden offiziell abgeschlossen.
6. Lessons Learned archivieren.
7. Historie niemals löschen.

---

# Change Policy

Änderungen erfolgen ausschließlich gemäß CHANGE_STANDARD.md.
