# PROMPT STANDARD
## Asphalt Burn™ Prompt Governance Standard

**Version:** 1.0.0  
**Status:** RELEASED  
**Classification:** GOVERNANCE  
**Owner:** Thomas Ridderskamp

---

# Purpose

Dieser Standard definiert die Erstellung, Verwaltung und Archivierung aller KI-Prompts innerhalb des Asphalt Burn™ Management Systems.

Prompts stellen einen zentralen Bestandteil des geistigen Eigentums der Marke dar und müssen jederzeit reproduzierbar, nachvollziehbar und versioniert sein.

---

# Scope

Dieser Standard gilt für:

- Bildgenerierung
- Textgenerierung
- Produktentwicklung
- Marketing
- SEO
- Social Media
- Dokumentation
- Automatisierung
- interne KI-Workflows

---

# Prompt Principles

Jeder Prompt muss:

- eindeutig formuliert
- reproduzierbar
- dokumentiert
- versioniert
- nachvollziehbar

sein.

---

# Mandatory Prompt Documentation

Für jeden produktionsrelevanten Prompt werden mindestens dokumentiert:

- Titel
- Zweck
- vollständiger Original-Prompt
- verwendetes KI-Modell
- Datum
- Version
- Ergebnis
- Freigabestatus

---

# Prompt Versioning

Prompts werden versioniert.

Beispiele:

v1.0.0

v1.1.0

v2.0.0

Frühere Versionen bleiben vollständig erhalten.

---

# Prompt Library

Alle Prompts werden zentral archiviert.

Es dürfen keine produktionsrelevanten Prompts ausschließlich in Chatverläufen verbleiben.

---

# Prompt Quality

Prompts sollen:

- eindeutig
- präzise
- vollständig
- testbar
- reproduzierbar

sein.

---

# Prompt Review

Vor produktiver Nutzung erfolgt eine Prüfung auf:

- Verständlichkeit
- Reproduzierbarkeit
- Ergebnisqualität
- rechtliche Sicherheit
- Markenkonformität

---

# Runtime Reference

Operative Promptverwaltung erfolgt innerhalb:

RUNTIME/

- PRODUCTS/
- PROJECTS/

---

# Governance Rules

1. Jeder Prompt wird archiviert.
2. Original-Prompts bleiben unverändert erhalten.
3. Änderungen werden versioniert.
4. Keine produktionsrelevanten Prompts verlieren.
5. Reproduzierbarkeit sicherstellen.
6. Ergebnisse dokumentieren.
7. Historie niemals löschen.

---

# Change Policy

Änderungen erfolgen ausschließlich gemäß CHANGE_STANDARD.md.
