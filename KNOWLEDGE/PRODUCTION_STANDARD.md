# PRODUCTION STANDARD
## Asphalt Burn™ Production Governance Standard

**Version:** 1.0.0  
**Status:** RELEASED  
**Classification:** GOVERNANCE  
**Owner:** Thomas Ridderskamp

---

# Purpose

Dieser Standard definiert sämtliche Anforderungen an die Herstellung aller Asphalt Burn™ Produkte.

Das Ziel ist eine gleichbleibend hohe Qualität, vollständige Reproduzierbarkeit und eine dokumentierte Produktionskette.

---

# Scope

Dieser Standard gilt für:

- Stickerei
- DTF-Druck
- Siebdruck
- Sublimation
- Patches
- Verpackung
- Qualitätskontrolle
- externe Produktionspartner

---

# Production Principles

Jede Produktion muss:

- reproduzierbar
- dokumentiert
- qualitätsgesichert
- wirtschaftlich
- rechtlich abgesichert

sein.

---

# Production Files

Vor Produktionsbeginn müssen vorhanden sein:

- Originaldesign
- Produktionsdateien
- Druckdateien
- Stickdateien (DST falls erforderlich)
- Farbdefinitionen
- Größenangaben
- Platzierungsdaten
- Produktionsfreigabe

---

# Embroidery Requirements

Für Stickprodukte gelten zusätzlich:

- saubere Vektorgrafik
- ausreichende Linienstärken
- stickgerechte Details
- dokumentierte Endgröße
- dokumentierte Stichart
- dokumentierte Garnfarben
- dokumentierte Stoffbasis

---

# DTF Requirements

Für DTF-Produkte gelten:

- transparenter Hintergrund
- ausreichende Auflösung
- scharfe Kanten
- RGB-Farbraum
- produktionstaugliche Dateiformate
- Freigabe nach Sichtkontrolle

---

# Supplier Verification

Jeder Produktionspartner muss dokumentiert werden.

Mindestens:

- Firmenname
- Ansprechpartner
- Produktionsverfahren
- Qualitätsbewertung
- bekannte Einschränkungen
- Reklamationshistorie

---

# Quality Control

Vor jeder Veröffentlichung erfolgt mindestens:

- Sichtprüfung
- Rechtsprüfung
- Druckprüfung
- Dateiprüfung
- Maßprüfung
- Freigabe durch den Owner

---

# Non-Conformities

Fehler werden dokumentiert.

Beispiele:

- falsche Farben
- beschädigte Ware
- Druckfehler
- Stickfehler
- Produktionsabweichungen
- Lieferfehler

---

# Documentation

Für jede Produktionscharge werden dokumentiert:

- Datum
- Lieferant
- Produkt
- Version
- Produktionsverfahren
- Ergebnis
- Auffälligkeiten

---

# Runtime Reference

Operative Produktionsdaten befinden sich ausschließlich innerhalb:

RUNTIME/

- PRODUCTS/
- SUPPLIERS/

---

# Governance Rules

1. Keine Produktion ohne Freigabe.
2. Keine Produktion ohne vollständige Dateien.
3. Jede Produktionsabweichung dokumentieren.
4. Jede Reklamation analysieren.
5. Qualitätsdaten niemals löschen.
6. Produktionsparameter versionieren.

---

# Change Policy

Änderungen erfolgen ausschließlich gemäß CHANGE_STANDARD.md.
